the loop

Four moves from blank canvas to a system under fire.

No YAML by hand, no docker-compose to babysit. The graph you draw is the spec — sydes compiles it, runs it, and feeds the numbers back.

01 / WIRE

Drag & connect

Drop services, an LB, Kafka, a worker, Postgres. Wire them up — only legal edges compile (default-deny).

02 / COMPILE

Graph → compose

The compiler emits a docker-compose file, an nginx upstream, and a tagged k6 script from your node config.

03 / RUN

Real containers

An isolated bridge network spins up per experiment. Kafka warms, healthchecks gate, the stack goes green.

04 / WATCH

Live metrics

Per-node CPU, memory and DB writes stream onto the canvas over WebSocket while k6 hammers your targets.

what's in the box

Not a diagram tool. A simulator with teeth.

Every box on the canvas is a process you can saturate. Every number is measured, not mocked.

// load

Per-service k6 targeting

Mark any service or LB as a load source with its own RPS. One tagged constant-arrival-rate scenario per target, per-target results, and a saturation highlight when achieved < target.

// telemetry

Live metrics over WebSocket

dockerode container.stats → CPU% and memory badges on every node, plus per-DB write counts — streamed straight to the canvas. No Prometheus, no cAdvisor.

// safety

Edge legality

A default-deny allowlist rejects dangling refs, self-loops and illegal pairs. The compiler fails loud, never best-effort.

// realism

Payload sensitivity

Give a service ms/KB and a source a body size. A fat payload raises p95 and tips a target into saturation a small one couldn't.

// runtime

Real infra, isolated

apache/kafka (KRaft), Postgres, nginx LB, and scratch-static Go images — each experiment on its own bridge network.

architecture

Agent-first. The browser draws; your machine runs.

A browser can't touch the Docker socket — so a local Node agent wraps the engine and exposes it over plain HTTP+JSON to the React canvas.
